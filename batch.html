※ Onboarding

Essa página cumpre o papel de informar novos colaboradores da área de tecnologia, incluindo desenvolvedores, designers, QA e PO's

Explicar qual o contexto, objetivo, etapas anteriores e times envolvidos. Caracterizar essa página, ajuda os desenvolvedores a entender quais foram os passos anteriores antes do produto ser desenvolvido e claro que, sempre levando em consideração alguns critérios de aceitação por exemplo:

    — Pesquisas qualitativas com o usuário (Não colocar o link do repostório)

    — Pesquisas quantitativas com o usuário (Não colocar o link do repostório)

O critério de não vincular o link dos repostórios ajudam a preservar a anonimidade do usuário. Para caracterizar essas etapas no handoff, é de suma importância apenas indicar no retângulo amarelo as etapas

※ Drafts

Rascunhos ou mais conhecidos pelos designers drafts são um local para você colocar suas ideias que podem ser projetadas em baixa e média fidelidade sem gastar muito tempo e claro um ótimo lugar para deixar as ideias salvas para o futuro.

※ Protótipos

A página de protótipos é ideal para os Designers colocar os protótipos de alta fidelidade.

Designers de multiplas senioridades ainda passam pela dificuldade de organizar seus protótipos, às vezes misturando componentes, rabiscos de protótipos, protótipos de alta. Como solução essa página de protótipos serve para isolar o protótipo de alta fidelidade, para manutenção e soma de novas funcionalidades

※ Workstation

Aqui é o holofote. A partir daqui desenvolvedores se guiarão através dessa página.

A página Workstation, é utilizada para listar tanto o protótipo de alta fidelidade, quando o protótipo estátipo. O desenvolvedor produz aquilo que está vendo e aquilo que enviaram. O protótipo estático facilita o entendimento de cada página ou funcionalidade dentro do projeto, ou seja, o dev não precisará ficar rodando o protótipo para construir a demanda solicitada

O protótipo de alta fidelidade, dá uma imersão maior para o desenvolvedor sobre quais pontos do aplicativo se interligam e suas micro micro interações.

※ Components

Essa página é utilizada para listar todos os componentes do Design System, útil para desenvolvedor visualizar sua interação de forma isolada.
